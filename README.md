# WanderLust

A full-stack web application for travel enthusiasts to discover, share, and review amazing destinations around the world. Built with Node.js, Express, MongoDB, and modern web technologies.

## Features

- **User Authentication**: Secure user registration and login system
- **Travel Listings**: Create, read, update, and delete travel destinations
- **Image Upload**: Cloudinary integration for image storage and management
- **Reviews System**: Users can leave reviews and ratings for listings
- **Responsive Design**: Mobile-friendly interface with Bootstrap styling
- **Flash Messages**: User feedback for actions and errors
- **Session Management**: Persistent user sessions with MongoDB store

## Tech Stack

### Backend
- **Node.js** - Runtime environment
- **Express.js** - Web framework
- **MongoDB** - NoSQL database
- **Mongoose** - ODM for MongoDB
- **Passport.js** - Authentication middleware
- **Joi** - Data validation
- **Multer** - File upload handling
- **Cloudinary** - Cloud image storage

### Frontend
- **EJS** - Templating engine
- **EJS-Mate** - Layout management
- **CSS** - Custom styling
- **JavaScript** - Client-side interactivity
- **Map Integration** - Location mapping functionality

Before running this application, make sure you have the following installed:

- Node.js (version 22.12.0 or higher)

- Git

## Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/mukesh-093/WanderLust.git
   cd WanderLust
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Environment Setup**
   - Create a `.env` file in the root directory
   - Add the following environment variables:
     ```
     NODE_ENV=development
     SECRET=your_secret_key_here
     CLOUD_NAME=your_cloudinary_cloud_name
     CLOUD_API_KEY=your_cloudinary_api_key
     CLOUD_API_SECRET=your_cloudinary_api_secret
     ATLASDB_URL=your_mongodb_atlas_connection_string
     ```

4. **Database Setup**
   - Ensure MongoDB is running locally or use MongoDB Atlas
   - The application will automatically connect using the ATLASDB_URL

5. **Initialize Database** (Optional)
   ```bash
   node init/index.js
   ```
   This will populate the database with sample data.

## Usage

1. **Start the application**
   ```bash
   npm start
   ```

2. **Open your browser**
   Navigate to `http://localhost:8080` (or the port specified in your environment)

3. **Explore the app**
   - Register a new account or login
   - Browse travel listings
   - Create your own listings
   - Leave reviews and ratings

## Database Models

### Listing Model
```javascript
{
  title: String (required),
  description: String,
  image: {
    url: String,
    filename: String
  },
  price: Number,
  location: String,
  country: String,
  reviews: [ObjectId] (references Review model),
  owner: ObjectId (references User model)
}
```
- **Features**: Automatic review deletion when listing is deleted
- **Relationships**: Has many reviews, belongs to one owner

### Review Model
```javascript
{
  comment: String,
  rating: Number (1-5),
  createdAt: Date (default: current timestamp),
  author: ObjectId (references User model)
}
```
- **Validation**: Rating must be between 1 and 5
- **Timestamps**: Automatic creation timestamp

### User Model
```javascript
{
  email: String (required),
  username: String (auto-generated by passport),
  password: String (hashed by passport)
}
```
- **Authentication**: Uses Passport-Local-Mongoose for username/password handling
- **Security**: Passwords are automatically hashed and salted

## Validation Schemas

### Listing Validation
- **Title**: Required string
- **Description**: Required string
- **Location**: Required string
- **Country**: Required string
- **Price**: Required number, minimum 0
- **Image**: Optional string (URL)

### Review Validation
- **Rating**: Required number between 1-5
- **Comment**: Required string

## Middleware Functions

### Authentication Middleware
- `isLoggedIn`: Ensures user is authenticated before accessing protected routes
- `saveRedirectUrl`: Preserves intended destination after login
- `isOwner`: Verifies user owns the listing before allowing modifications

### Validation Middleware
- `validateListing`: Validates listing data using Joi schema
- `validateReview`: Validates review data using Joi schema

### Error Handling
- Custom `ExpressError` class for consistent error responses
- `wrapAsync` utility for handling async route errors
- Centralized error handling in main app.js

## Troubleshooting

### Common Issues

**Database Connection Failed**
- Check MongoDB Atlas connection string
- Verify network connectivity
- Ensure IP whitelist includes your IP

**Image Upload Not Working**
- Verify Cloudinary credentials
- Check file size limits
- Ensure proper file formats (JPEG, PNG, JPG)

**Authentication Issues**
- Clear browser cookies and session storage
- Check SECRET environment variable
- Verify MongoDB session store connection

**Port Already in Use**
- Change PORT environment variable
- Kill process using the port: `npx kill-port 8080`

### Debug Mode
Set `NODE_ENV=development` for detailed error messages and stack traces.
